AI INTERACTION DESIGN

Simplifying AI agent setup for an enterprise Voice AI platform through a focused UX audit

4-hour audit with ready-to-dev improvements
View case study deck ↗

CHALLENGE

Synthflow offers AI agent capabilities for voice-based workflows, but the setup experience exposed prompts, actions, variables, and settings all at once. For first-time users, this created cognitive overload and made the product feel architecture-driven rather than goal-driven.

MY TASK

Conduct a fast UX audit of the current setup experience, identify the highest-friction points, and propose a practical redesign direction that could improve clarity and usability without requiring major logic and backend changes.

TASK TIMEFRAME

4 hours

KEY FOCUS AREAS

  • Internal UX audit through first-time user simulation
  • Audience and persona framing for AI agent setup
  • Prompt editor friction analysis
  • Goal-driven UX restructuring
  • Low-effort prototype for frontend-only improvements

KEY RESULTS

  • Identified 3 core friction points in the current setup flow: cognitive overload, architecture-driven UX, and premature feature exposure.
  • Designed a frontend-only restructuring concept focused on: clearer hierarchy, structured prompt editing, and better separation between core task and advanced configuration.
  • Proposed a solution that keeps existing backend logic intact while making the setup flow easier to understand and faster to complete for less technical users.

OVERVIEW OF MY DESIGN PROCESS

Step 1 — Framing the audience and business context

Mapped the platform’s likely audience across four levels of technical familiarity: non-technical users, prompt designers, automation builders, and developers. Then narrowed the audit to a realistic business use case: real estate lead qualification, where an AI agent helps qualify inbound leads before a realtor follows up.

Step 2 — Defining a realistic persona and goal

Grounded the audit in a real-world persona based on previous interviews with U.S. realtors. I used “Jay,” a semi-technical realtor familiar with ChatGPT, Calendly, and basic automation, whose goal was simple: set up an AI agent to contact leads, ask qualification questions, collect property preferences, and identify which leads were worth his time.

Step 3 — Simulating the first-time user flow

Explored the platform step by step as a new user:

  • created an agent using a template
  • edited the prompt
  • explored custom actions
  • tested the agent via chat
  • reviewed the conversation results

This revealed where the product flow broke down for users trying to complete a simple task.

Step 4 — Identifying the 3 core UX problems

The audit surfaced three systemic problems:

  • 1. Cognitive overload: The user had to understand multiple systems at once: prompt, settings, and actions.
  • 2. Architecture-driven UX: The interface reflected internal system structure instead of guiding users toward their goal.
  • 3. Premature feature exposure: The platform exposed advanced options before users understood their value.

Step 5 — Exploring broader redesign directions

I briefly explored larger product opportunities, such as:

  • turning raw extractors into goal-based actions
  • restructuring setup into smaller guided steps
  • using more user-friendly naming for actions and variables

But because the exercise was time-constrained, I deliberately narrowed the final solution to one practical change with minimal implementation cost.

Step 6 — Designing a focused frontend solution

I redesigned the main setup screen around a clearer mental model:

Agent Behavior on the left

  • greeting message
  • background info
  • agent info
  • objection handling
  • call script instructions

Agent Configuration on the right

  • actions
  • variables
  • version history
  • global settings
  • deployment

I replaced the intimidating prompt wall with a structured, form-like layout while still preserving access to raw prompt editing for more advanced users.

CORE RESULT

The final concept showed how Synthflow could move from a fragmented, system-oriented setup experience to a more guided and goal-driven flow without changing the backend logic.

The main improvement was not a full redesign, but a better structure:

  • less cognitive overload
  • clearer setup hierarchy
  • easier understanding for first-time users
  • preserved flexibility for technical users

NEXT STEPS

If expanded beyond the 4-hour audit, I would validate the concept through:

  • time-to-first-successful-setup measurement
  • drop-off analysis during setup
  • number of test runs before successful publish
  • heatmap/session recordings
  • quick comprehension testing: Can a user clearly explain what their agent does after setup?
