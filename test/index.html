2. Problem framing

Two compounding challenges

Scattered data in legacy workflows

Critical deal intelligence lived across PDFs, Excel models, and disconnected BI tools. Extracting and comparing data was heavy manual work. Historical context was regularly lost — too hard to structure over time.

AI hallucination risk in finance

Even one incorrectly generated rent number would destroy platform credibility. Users needed a transparent way to verify and correct AI outputs without breaking their existing workflow.

3. Strategy & trade-offs

From static data repository to automated workflows

When I joined in 2024, GoCanopy was a simple data repository with basic AI chat on top, targeting investors only. As the product expanded, I helped reframe the direction toward automating complex workflows — like generating underwriting reports and actionable insights.

The core tension: automation vs. control

One early hypothesis was full agent automation — trigger underwriting, and background agents independently generate a final report. But interviews with existing clients revealed a hard truth: analysts were deeply wary of AI in finance. One hallucinated number could ruin a deal. This was already visible in the existing platform through source-linking components designed to help users verify data.

UX pivot: from fully autonomous agents → co-creation model

Through alignment with engineering, I advocated that a pure "black box" workflow was too fragile for enterprise finance. I designed the co-creator approach: instead of generating a full document in one step, AI produces smaller modular outputs — tables, assumptions, insights — that users can edit, regenerate, trace to source data, and compose themselves.

I also designed inline editing: if AI generated an incorrect cap rate, users could edit it directly, and the system automatically recalculated the rest of the model.

For AI deal extraction, I designed a clear validation flow: newly extracted data was marked "Under review" with subtle visual cues, and only transitioned to "Verified" after user confirmation or correction.

4. Design execution

Five steps from concept to prototype

1

Research & new asset management logic

Mapped core underwriting workflows to define how AI document extraction, human review loops, and artifact generation would interact. Defining this system logic upfront was essential to handle complex data hierarchies (Deals → Assets → Leases) without dead ends or broken mental models.

2

Vibe-coding early concepts

To hit the 2-month deadline, I rapidly validated code-based layouts with the CTO using Generative UI tools like V0.dev and Claude Code, then moved to Figma to refine the interactions that would make or break user adoption.

3

From wireframing to high-fidelity prototype

Started with core data requirements in early wireframes, adjusting layouts as we received actual BI dashboard examples from clients. Experimented with conversational UIs and embedded AI insights. After testing with engineering, shifted to the co-creator workspace model using step-controlled agents. Then applied the design system to build the final high-fidelity prototype.

4

Design system foundation

The platform already had baseline styling. I audited and structured existing styles into a formal design system, then optimised components for enterprise workflows: high-density typography, strict table behaviours, and clear color semantics (specific colors reserved exclusively for AI-generated vs. human-verified data).

5

Three high-risk workflows for the enterprise pitch

Automated document ingestion: Replaced passive upload with a drag-and-drop zone that instantly generates deal cards and shows extraction progress — giving users immediate visibility into AI status.

Verifiable source linking: To counter AI hallucination risk, the UI connects extracted metrics to their origin. Clicking a data point splits the screen and highlights the exact paragraph in the source PDF, enabling instant auditability.

Proactive insights & alerts: Instead of forcing users to search for risks in tables, the system automatically flags upcoming lease breaks and visualises rent positioning against market comps directly on the deal overview.

6. Learnings

What I'd take to the next project

Friction builds trust

UX usually preaches frictionless flows. But in high-stakes finance, seamless AI feels suspicious. Forcing users to verify source links and manually approve "Under review" data added the exact friction needed to win Investment Committee trust.

Match UI to technical reality

My initial design was sequential: upload a deal, click to trigger AI analysis. After reviewing with the CTO, we realised some agents could run automatically in the background right after extraction. Redesigning to show proactive insights upfront was a much better experience — anticipating what users needed and delivering it instantly.

The cost of pitch-speed design

Moving fast meant shortcuts. I didn't create a consistent system for data charts upfront. When we needed to add new chart types for different assets, it required messy rework. Next time: define the rules for all charts in week one, even if we don't build them all immediately.
